---
title: "Revoke API Key"
api: "DELETE /api/v1/api-keys/{id}"
description: "Revoke an API key by setting its revoked_at timestamp. The key can no longer be used for authentication."
---

## Authentication

Requires Supabase session authentication (dashboard users only).

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the API key to revoke (prefixed with `ak_`)
</ParamField>

## Request

No request body required.

## Response

<ResponseField name="status" type="string" required>
  Always returns `"revoked"` on success
</ResponseField>

## Example

```bash
curl -X DELETE https://buildstory-cli.vercel.app/api/v1/api-keys/ak_a1b2c3d4e5f6g7h8 \
  -H "Cookie: sb-access-token=YOUR_SESSION_TOKEN"
```

### Response

```json
{
  "status": "revoked"
}
```

## Behavior

- **Soft delete**: The key is not deleted from the database - instead, the `revoked_at` field is set to the current timestamp
- **Immediate effect**: Once revoked, the key can no longer be used for authentication
- **Idempotent**: Attempting to revoke an already-revoked key returns a 404 error
- **Ownership check**: Users can only revoke their own keys - attempting to revoke another user's key returns 404

## Error Responses

<ResponseField name="error" type="string">
  Error message
</ResponseField>

### 401 Unauthorized

Returned when the request is not authenticated with a valid Supabase session.

```json
{
  "error": "Unauthorized"
}
```

### 404 Not Found

Returned when:
- The key ID does not exist
- The key belongs to a different user
- The key is already revoked

```json
{
  "error": "Key not found"
}
```
