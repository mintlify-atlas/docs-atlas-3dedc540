---
title: "Create API Key"
api: "POST /api/v1/api-keys"
description: "Generate a new API key for authentication. Automatically revokes all existing active keys for the user."
---

## Authentication

Requires Supabase session authentication (dashboard users only).

## Request

No request body required.

## Response

<ResponseField name="id" type="string" required>
  The unique identifier for the API key (prefixed with `ak_`)
</ResponseField>

<ResponseField name="key" type="string" required>
  The full API key value (prefixed with `bsk_`). **This is only returned once** - store it securely.
</ResponseField>

<ResponseField name="key_prefix" type="string" required>
  The first 8 characters of the key for display purposes (e.g., `bsk_a1b2`)
</ResponseField>

<ResponseField name="label" type="string" required>
  The label for this key (default: "Default")
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp when the key was created
</ResponseField>

## Example

```bash
curl -X POST https://buildstory-cli.vercel.app/api/v1/api-keys \
  -H "Cookie: sb-access-token=YOUR_SESSION_TOKEN"
```

### Response

```json
{
  "id": "ak_a1b2c3d4e5f6g7h8",
  "key": "bsk_1234567890abcdef1234567890abcdef",
  "key_prefix": "bsk_1234",
  "label": "Default",
  "created_at": "2026-03-03T12:00:00.000Z"
}
```

## Behavior

- **Revokes existing keys**: All active API keys for the authenticated user are automatically revoked before generating the new key
- **Single active key**: This ensures each user has only one active API key at a time
- **One-time display**: The full `key` value is only returned in this response - it cannot be retrieved later
- **Prefix format**: All API keys use the `bsk_` prefix followed by 32 hexadecimal characters

## Error Responses

<ResponseField name="error" type="string">
  Error message
</ResponseField>

### 401 Unauthorized

Returned when the request is not authenticated with a valid Supabase session.

```json
{
  "error": "Unauthorized"
}
```

### 500 Internal Server Error

Returned when key generation fails.

```json
{
  "error": "Internal server error"
}
```
