---
title: "Dismiss Draft Update"
api: "POST /api/v1/updates/{entryId}/dismiss"
description: "Dismiss a draft update to exclude it from the public timeline"
---

## Authentication

Requires Supabase session authentication (cookie-based). User must own the project this update belongs to.

## Path Parameters

<ParamField path="entryId" type="string" required>
  Entry ID of the draft to dismiss (format: `e_*`)
</ParamField>

## Request Body

No request body required.

## Response

<ResponseField name="status" type="string">
  Returns "ok" on success
</ResponseField>

<RequestExample>
```bash
curl -X POST https://buildstory-cli.vercel.app/api/v1/updates/e_xyz789/dismiss \
  -H "Cookie: your-session-cookie"
```
</RequestExample>

<ResponseExample>
```json
{
  "status": "ok"
}
```
</ResponseExample>

## Behavior

- Sets the update status to `"dismissed"`
- Updates the `updated_at` timestamp
- Removes the update from the draft queue
- Prevents the update from appearing on the public timeline

## Error Responses

<ResponseField name="error" type="string">
  Error message
</ResponseField>

| Status Code | Error Message | Description |
|-------------|---------------|-------------|
| 401 | "Unauthorized" | No valid session |
| 404 | "Draft not found or already processed" | Entry doesn't exist or is no longer in draft status |
| 500 | "Internal server error" | Database or server error |

## Use Cases

- Hide minor commits (typo fixes, formatting changes)
- Exclude work-in-progress commits
- Remove updates that don't represent meaningful progress
- Filter out commits that were later reverted
