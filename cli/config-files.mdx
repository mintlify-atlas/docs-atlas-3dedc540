## Overview

When you run `buildstory init` in a git repository, it creates a `.buildstory/` directory containing project-specific configuration and logs. This directory is automatically added to `.gitignore` and should not be committed.

## Directory Structure

```
.buildstory/
├── config.json    # Project configuration
└── hook.log       # Post-commit hook activity log
```

## config.json

The project configuration file links your repository to a specific Buildstory project.

### Location

```
.buildstory/config.json
```

### File Structure

```json
{
  "project_id": "proj_a1b2c3d4e5f6",
  "endpoint": "https://buildstory-cli.vercel.app",
  "machine_id": "mycomputer-abc123",
  "created_at": "2024-03-15T10:30:00.000Z"
}
```

### Fields

<ParamField path="project_id" type="string" required>
  Unique identifier for your Buildstory project. Generated by the server during initialization. Starts with the `proj_` prefix.
</ParamField>

<ParamField path="endpoint" type="string" required>
  API endpoint URL for the Buildstory server. Defaults to `https://buildstory-cli.vercel.app` but can be customized with the `-e` flag during init.
</ParamField>

<ParamField path="machine_id" type="string" required>
  Unique identifier for this machine, combining your hostname with a random suffix. Used to distinguish commits from different development environments. Format: `hostname-randomhex`.
</ParamField>

<ParamField path="created_at" type="string" required>
  ISO 8601 timestamp indicating when the project was initialized in this repository.
</ParamField>

### How It's Used

The post-commit hook reads this file to:
- Determine which project to send commits to
- Identify the API endpoint
- Tag commits with the machine ID for environment tracking

If this file is missing, the hook will log a SKIP entry and exit silently.

### Custom Endpoint

You can specify a custom API endpoint during initialization:

```bash
buildstory init --endpoint https://your-custom-endpoint.com
```

Or set the `BUILDSTORY_ENDPOINT` environment variable:

```bash
export BUILDSTORY_ENDPOINT=https://your-custom-endpoint.com
buildstory init
```

## hook.log

The hook log file records every execution of the post-commit hook, providing visibility into commit delivery.

### Location

```
.buildstory/hook.log
```

### Log Format

Each line follows this format:

```
YYYY-MM-DDTHH:MM:SSZ [STATUS] commit_hash details
```

### Status Types

<ParamField path="OK" type="status">
  Commit data was successfully sent to the server (HTTP 2xx response).
  
  **Example:**
  ```
  2024-03-15T10:30:00Z [OK] abc123 entry=e_1234567890abcdef http=200
  ```
</ParamField>

<ParamField path="FAIL" type="status">
  Failed to send commit data to the server (HTTP error or network failure).
  
  **Example:**
  ```
  2024-03-15T10:30:00Z [FAIL] abc123 entry=e_1234567890abcdef http=500 body=Internal server error
  ```
</ParamField>

<ParamField path="SKIP" type="status">
  Hook execution was skipped due to missing configuration or credentials.
  
  **Examples:**
  ```
  2024-03-15T10:30:00Z [SKIP] abc123 config_missing path=.buildstory/config.json
  2024-03-15T10:30:00Z [SKIP] abc123 creds_missing path=/home/user/.buildstory/credentials.json
  2024-03-15T10:30:00Z [SKIP] abc123 empty_config project_id=<empty> endpoint=<empty> token=set
  ```
</ParamField>

### Log Rotation

The hook log automatically rotates when it exceeds ~100KB:
- When the file reaches this size, it's truncated to the last 50KB
- This prevents unbounded log growth while preserving recent history
- No manual log management is required

### Checking Hook Status

Use `buildstory status` to get a structured summary of hook activity:

```bash
buildstory status
```

This command analyzes `hook.log` and reports:
- Total OK/FAIL/SKIP counts
- Recent failures with details
- Commit delivery verification (compares local commits with server records)

### Common SKIP Reasons

1. **config_missing**: The `.buildstory/config.json` file doesn't exist
   - **Fix**: Run `buildstory init` in the repository

2. **creds_missing**: The `~/.buildstory/credentials.json` file doesn't exist
   - **Fix**: Run `buildstory auth` to authenticate

3. **empty_config**: The config file exists but contains empty/invalid values
   - **Fix**: Delete `.buildstory/config.json` and run `buildstory init` again

## .gitignore Entry

During `buildstory init`, the entire `.buildstory/` directory is automatically added to `.gitignore`:

```gitignore
.buildstory/
```

This ensures that project configuration and logs remain local to your machine and are never committed to version control.

## Security

<Warning>
  While the `.buildstory/` directory doesn't contain authentication tokens (those are in `~/.buildstory/credentials.json`), it does contain your project ID and machine ID. Keep this directory out of version control.
</Warning>

- The directory is automatically gitignored during initialization
- Project IDs are not sensitive but link commits to your Buildstory project
- Machine IDs help track which development environment made each commit
