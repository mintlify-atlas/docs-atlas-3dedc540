## Overview

The credentials file stores your Buildstory authentication token globally on your machine. It is created when you run `buildstory auth` and is used by all Buildstory-enabled repositories on your system.

## Location

```
~/.buildstory/credentials.json
```

The file is stored in your home directory and is shared across all projects.

## File Structure

```json
{
  "token": "bsk_1a2b3c4d5e6f7g8h9i0j...",
  "created_at": "2024-03-15T10:30:00.000Z"
}
```

## Fields

<ParamField path="token" type="string" required>
  Your Buildstory API key. Generated during authentication and starts with the `bsk_` prefix.
</ParamField>

<ParamField path="created_at" type="string" required>
  ISO 8601 timestamp indicating when the credentials were created.
</ParamField>

## Usage

### Authentication

The credentials file is automatically created when you authenticate:

```bash
buildstory auth
```

This command opens your browser for authentication and saves the API key to the credentials file.

### Manual Token Entry

You can also provide a token directly (useful for CI/scripting):

```bash
buildstory auth --token bsk_your_api_key_here
```

### How It's Used

The post-commit hook reads this file to authenticate API requests when sending commit data to the Buildstory server. The token is sent as a Bearer token in the `Authorization` header:

```bash
Authorization: Bearer bsk_1a2b3c4d5e6f7g8h9i0j...
```

## Security

<Warning>
  The credentials file contains sensitive authentication data. Never commit it to version control or share it publicly.
</Warning>

- The file is stored in your home directory, outside of any git repository
- File permissions should be readable only by your user account
- The token provides full access to your Buildstory account
- If compromised, revoke the token from your dashboard and run `buildstory auth` again

## Troubleshooting

### Missing Credentials

If the post-commit hook cannot find the credentials file, it will log a SKIP entry:

```
2024-03-15T10:30:00Z [SKIP] abc123 creds_missing path=/home/user/.buildstory/credentials.json
```

To resolve this, run `buildstory auth` to create the credentials file.

### Invalid Token

If your token is invalid or expired, API requests will fail with a 401 status. Check `.buildstory/hook.log` for FAIL entries:

```
2024-03-15T10:30:00Z [FAIL] abc123 entry=e_1234567890abcdef http=401 body=...
```

Reauthenticate by running `buildstory auth` again.
