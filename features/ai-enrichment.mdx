Buildstory uses AI to automatically transform raw git commit data into clear, structured project updates that tell the story of what you're building.

## How It Works

Every time you commit code in a connected repository, the git post-commit hook sends commit metadata to Buildstory's server. The enrichment process then runs automatically:

1. **Raw commit data is captured** - commit message, file changes, diff stats, and environment context
2. **AI analyzes the commit** - using Anthropic's Claude to understand what actually changed
3. **Structured update is created** - title, description, type, tags, and toolchain detection
4. **Draft appears in review queue** - ready for you to approve, edit, or dismiss

## AI Model and Approach

Buildstory uses **Claude Sonnet 4.5** (`claude-sonnet-4-5-20250929`) via the Anthropic SDK.

The enrichment process uses Anthropic's **tool-use pattern** to generate structured output. Instead of parsing unstructured text, Claude is given a tool definition called `create_update` with a strict schema, ensuring consistent results every time.

### What the AI receives

The AI analyzes:
- Commit message and author
- Diff statistics (files changed, insertions, deletions)
- List of changed files with change types (added, modified, deleted)
- Diff summary (if available)
- Detected development environment (Cursor, VS Code, terminal)

### What the AI produces

Each enriched update includes:

<ResponseField name="title" type="string" required>
  A clear, specific title (5-12 words) describing what was done, not just restating the commit message
</ResponseField>

<ResponseField name="description" type="string" required>
  1-3 sentences explaining what changed, why it matters, and notable technical details
</ResponseField>

<ResponseField name="type" type="enum" required>
  One of: `feature`, `fix`, `improvement`, `refactor`, `setup`, `research`
</ResponseField>

<ResponseField name="tags" type="array" required>
  2-5 tags extracted from file paths, technologies, or concepts (e.g., "api", "auth", "react")
</ResponseField>

<ResponseField name="toolchain.tool_detected" type="string">
  Detected framework or tool (e.g., "Next.js", "Tailwind", "Prisma")
</ResponseField>

<ResponseField name="toolchain.model_used" type="string">
  AI model identifier (always `claude-sonnet-4-5-20250929`)
</ResponseField>

## Update Types

The AI categorizes each commit into one of six types:

<AccordionGroup>
  <Accordion title="Feature">
    New functionality added to the project. This is for wholly new capabilities, not enhancements to existing features.
  </Accordion>
  
  <Accordion title="Fix">
    Bug fixes and corrections to existing functionality.
  </Accordion>
  
  <Accordion title="Improvement">
    Enhancements to existing features. Performance optimizations, UX improvements, or refinements to current functionality.
  </Accordion>
  
  <Accordion title="Refactor">
    Code restructuring without changing external behavior. Reorganizing files, renaming variables, or cleaning up code.
  </Accordion>
  
  <Accordion title="Setup">
    Project configuration, dependencies, CI/CD, build tools, or infrastructure changes. Anything touching package.json, config files, or tooling.
  </Accordion>
  
  <Accordion title="Research">
    Exploration, experiments, or spike work. Temporary code to test ideas or investigate approaches.
  </Accordion>
</AccordionGroup>

## Enrichment Pipeline

The enrichment process happens in two stages:

### 1. Immediate enrichment (after commit)

When a commit is received at `/api/v1/ingest`, the server responds immediately to your git hook, then triggers enrichment in the background using Next.js's `after()` hook. This ensures your commit process isn't blocked waiting for AI.

### 2. Sweep job (every 15 minutes)

A cron job at `/api/cron/enrich` runs every 15 minutes to catch any commits that failed enrichment or were missed. It processes up to 10 unenriched entries per run.

<Note>
  Entries that fail enrichment 3 times are marked as "poison pills" and won't be retried automatically. This prevents the same broken commit from blocking the queue.
</Note>

## Manual Enrichment

If commits are stuck waiting for enrichment, you can trigger manual enrichment from the project dashboard:

1. Go to your project page in the dashboard
2. Look for the **Enrichment Status** panel
3. Click **Enrich Now** to manually process up to 10 pending commits

This is useful when:
- The automatic enrichment failed due to a temporary API issue
- You want to see drafts immediately instead of waiting for the cron job
- Commits are pending for more than 2 minutes

## Editing AI Results

AI-generated updates appear as **drafts** in your review queue. You can:

- **Edit the title** - Click the title to edit inline
- **Edit the description** - Click the description to edit inline  
- **Add a decision note** - Optional context about why you approved this update
- **Approve** - Move to your public timeline
- **Dismiss** - Hide from timeline (won't appear publicly)

Edits are saved immediately when you click outside the field or press Enter.

## Privacy and Content Filtering

The git hook automatically redacts common secret patterns from diff summaries before sending to the server. Environment variables, API keys, and tokens are filtered out.

You can also use `.buildstoryignore` to exclude entire files or directories from being sent:

```
.env*
secrets/
config/credentials.yml
```

<Warning>
  While the hook filters common patterns, always review drafts before approving them to your public timeline. Never approve updates that contain credentials, private keys, or sensitive business logic.
</Warning>
